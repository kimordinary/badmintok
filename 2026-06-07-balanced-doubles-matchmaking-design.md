# 배드민턴 복식 균형 대진표 생성 알고리즘 — 설계 문서

작성일: 2026-06-07

## 1. 목적

급수가 다양한 참가자들이 **균형 있고 공정하게** 배드민턴 복식 경기를 할 수 있도록
대진표(라운드별 경기 목록)를 자동 생성하는 알고리즘을 정의한다.

핵심 목표 세 가지:

1. **팀 간 실력 균형** — 한 경기에서 맞붙는 두 팀의 실력 합이 비슷할 것 (경기 자체가 박빙)
2. **출전 공정성** — 모든 참가자가 비슷한 횟수로 경기에 참여할 것
3. **다양성** — 같은 파트너·같은 상대와의 반복 매칭을 최소화할 것

여기에 더해 **성별·종목(남복·여복·혼복)** 을 팀 구성의 주요 지표로 반영한다.

## 2. 급수와 점수 환산

배드민턴 급수는 다음 7단계로 정의한다 (높을수록 실력 상위):

| 급수 | 자강 | S | A | B | C | D | E(초심) |
|------|------|---|---|---|---|---|---------|
| 점수 | 7 | 6 | 5 | 4 | 3 | 2 | 1 |

- **균등 간격(1~7)** 으로 환산한다.
- 이 점수를 **기본 점수(base score)** 라 한다.

### 2.1 종목별 유효 점수 (effective score)

실제 매칭 계산에는 기본 점수가 아니라 **종목에 따라 보정된 유효 점수**를 사용한다.

- **남복 / 여복**: 같은 성별끼리 경기하므로 보정하지 않는다. 유효 점수 = 기본 점수.
- **혼복**: 일반적으로 동일 급수에서 여자의 복식 기여도가 남자보다 낮게 평가되는 관행을 반영한다.
  - 여자 유효 점수 = `기본 점수 − femaleAdjust`
  - 남자 유효 점수 = `기본 점수`
  - `femaleAdjust`는 설정값(기본 1). 즉 혼복에서 여자는 한 급수 낮게 취급된다.
  - 하한 클램프: 유효 점수가 1 미만이 되지 않도록 `max(1, 기본 점수 − femaleAdjust)`.

> 보정값을 설정값으로 분리해 두면, 운영 결과를 보고 혼복의 남녀 가중을 조절할 수 있다.

## 3. 입력 / 출력

### 3.1 입력

| 항목 | 설명 |
|------|------|
| 참가자 목록 | 각 참가자는 `{이름, 급수, 성별}` |
| 코트 수 `C` | 동시 진행 가능한 경기 수 |
| 라운드 수 `R` | 또는 "1인당 목표 경기 수" (둘 중 하나로 환산) |
| 종목별 비율 | 혼복 / 여복 / 남복의 비율 (예: 60% / 25% / 15%) |
| `femaleAdjust` | 혼복 여자 점수 차감값 (기본 1) |
| 가중치 | `w_balance`, `w_partner`, `w_opponent`, `w_fairness` |

### 3.2 출력

| 항목 | 설명 |
|------|------|
| 라운드별 경기 목록 | `라운드 → [코트별 (종목, 팀1 2명) vs (팀2 2명)]` |
| 라운드별 휴식 인원 | 해당 라운드에 출전하지 않은 참가자 |
| 요약 통계 | 1인당 경기 수(최소/최대/평균), 평균 팀 간 실력차, 파트너·상대 중복 횟수, 종목 비율 달성률, 경고 메시지 |

## 4. 누적 상태 추적

라운드를 진행하며 다음 상태를 계속 갱신한다. 이 상태가 공정성·다양성 판단의 근거가 된다.

- `played[p]` : 참가자 `p`의 누적 출전 횟수
- `partnerCount[p][q]` : `p`와 `q`가 **같은 팀**이었던 횟수
- `opponentCount[p][q]` : `p`와 `q`가 **상대 팀**이었던 횟수

## 5. 비용 함수

하나의 경기(4명, 두 팀)가 얼마나 "나쁜 매칭"인지를 단일 점수로 평가한다.
점수가 **낮을수록 좋은** 매칭이다.

```
cost(team1, team2) =
      w_balance  × (유효점수합(team1) − 유효점수합(team2))²
    + w_partner  × (partnerCount[team1쌍] + partnerCount[team2쌍])
    + w_opponent × (team1 × team2 상대쌍들의 opponentCount 합)
    + w_fairness × (네 명의 played 합에 비례하는 가산)
```

설계 의도:

- **실력차는 제곱항** — 작은 불균형은 너그럽게, 큰 불균형은 강하게 회피한다.
- **파트너 중복** — 이미 같은 팀이었던 쌍을 다시 묶으면 페널티.
- **상대 중복** — 이미 맞붙었던 상대끼리 또 만나면 페널티.
- **공정성** — 이미 많이 뛴 사람을 또 넣을수록 비용 가산 (출전 균등 유도).

라운드 전체 비용은 그 라운드 모든 경기의 `cost` 합이며, 알고리즘은 이 합을 줄이는 방향으로 동작한다.

### 5.1 가중치 운영

가중치는 운영자가 우선순위를 조절하는 손잡이다. 기본값은 **팀 간 균형 우선**으로 둔다.

- 균형을 더 중시 → `w_balance` ↑
- 다양한 사람과 치게 하려면 → `w_partner`, `w_opponent` ↑
- 출전 횟수 형평을 더 엄격히 → `w_fairness` ↑

## 6. 생성 알고리즘 (라운드 단위 반복)

각 라운드마다 다음 4단계를 수행한다.

### 6.1 코트별 종목 배정

- 라운드의 코트 `C`개를 **종목별 비율**에 따라 혼복 / 여복 / 남복으로 분배한다.
- 비율을 정수 코트 수로 환산할 때의 반올림 오차는 누적 보정하여, 여러 라운드에 걸쳐 목표 비율에 수렴하도록 한다.

### 6.2 종목 제약 풀 선정

- 각 코트의 종목에 맞는 성별 인원을 `played`가 **적은 순**으로 선발한다 (출전 공정성 확보).
  - **혼복 코트**: 기본 남자 2명 · 여자 2명 (팀당 남1·여1).
  - **여복 / 남복 코트**: 해당 성별 4명.
- 선발되지 않은 인원은 그 라운드 휴식. 휴식도 `played` 기준으로 공평하게 순환된다.
- 성별 인원이 종목 구성을 채우지 못하면 6.5(엣지 케이스) 규칙으로 처리한다.

### 6.3 그리디 초기 배치

- 선발된 인원을 **유효 점수순으로 정렬**한다.
- 코트마다 4명을 묶고, 팀 내 균형이 맞도록 (정렬 순위 기준) 1·4 / 2·3 형태로 두 팀을 구성한다.
  - 즉 코트에 모인 4명 중 가장 강한 사람과 가장 약한 사람을 한 팀으로 묶어 팀 간 실력 합을 비슷하게 시작한다.
- 혼복 코트는 각 팀이 남1·여1이 되도록 구성한다.

### 6.4 지역 탐색 개선 (local search)

- 두 참가자를 **교환(swap)** 했을 때 라운드 전체 비용 합이 줄어들면 교환을 채택한다.
- 교환은 **같은 종목·같은 성별 슬롯 내에서만** 허용한다 (종목·성별 규칙을 깨지 않기 위함).
  - 예: 혼복 코트의 여자 슬롯끼리, 또는 남자 슬롯끼리만 교환.
- 더 이상 개선되는 교환이 없거나 최대 반복 횟수 `N`에 도달하면 종료한다.

### 6.5 상태 갱신

- 확정된 매칭으로 `played`, `partnerCount`, `opponentCount`를 갱신한다.
- 다음 라운드는 갱신된 상태를 바탕으로 6.1부터 다시 시작한다.

## 7. 엣지 케이스

| 상황 | 처리 |
|------|------|
| 인원이 4의 배수가 아님 | 매 라운드 휴식 인원 발생. `played` 기준으로 공평하게 순환 |
| 인원 < 4 | 경기 생성 불가. 사용자에게 안내 |
| 코트 수 > 가능 경기 수 | 실제 채울 수 있는 코트만 사용 |
| 특정 급수에 인원 집중 | 완벽한 균형이 불가하면 비용 함수가 자동으로 최소 불균형 매칭을 선택 |
| 성별 인원이 종목 비율을 못 채움 (예: 여자 부족) | 채울 수 없는 종목 코트의 비율을 자동 하향 조정하고 가능한 종목으로 대체. 요약에 경고 표시 |
| 혼복인데 남녀 수 불균형 | "유연 허용" 규칙에 따라 동성 페어 일부를 허용하되, 비용 함수로 발생을 최소화 |

## 8. 품질 검증 지표

생성 결과의 품질은 출력 요약 통계로 확인한다.

- **출전 균등성**: `played`의 최대−최소 차이가 작을수록 좋음
- **팀 간 균형**: 평균 |팀 점수합 차이|가 작을수록 좋음
- **다양성**: 파트너·상대 중복 횟수가 적을수록 좋음
- **종목 비율 달성률**: 실제 생성된 종목 분포가 목표 비율에 근접할수록 좋음

이 지표들은 가중치를 튜닝하는 피드백 루프로도 사용된다.

## 9. 확장 여지 (현재 범위 외)

- 시간대별 코트 배정·대기 순서까지 포함한 타임테이블 생성
- 부상·중도 이탈 참가자 실시간 재편성
- 과거 대회 이력을 누적해 장기적 다양성 확보
- 점수 환산 테이블·가중치의 운영자 UI 노출
