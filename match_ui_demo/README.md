# Handoff: 번개 실시간 자동 대진 운영 (Lightning Real-time Auto-Matchmaking)

## Overview
배드민턴 동호회 "번개(즉석 모임)"의 **실시간 자동 대진 운영 도구**입니다. 모임장(운영자)이 현장에서 태블릿/스마트폰으로 보며, 코트가 빌 때마다 시스템이 추천한 다음 4명(두 팀)을 진행시킵니다.

핵심 가치: **"처음 온 사람도 쉬지 않고 박빙 경기를."**
- 아무도 소외되지 않게 균등 출전 / 덜 뛴 사람 우선 / 팀 균형(박빙) / 파트너·상대 다양성 / 성별·종목(혼복·남복·여복) 반영

설계 원문은 프로젝트에 첨부된 설계 문서(`번개 실시간 자동 대진 — 설계 문서`)를 따릅니다.

> **운영자 루프(핵심 동선):** 경기 끝남 → `경기 종료` 한 번 누름 → 시스템이 다음 4명을 그 코트에 **자동 배정** + 하단에 **호출 토스트**(이름들) 표시 → 운영자가 호명 → 반복. 평상시 운영 액션은 사실상 "경기 종료" 하나.

## About the Design Files
이 번들의 파일들은 **HTML로 만든 디자인 레퍼런스**입니다 — 의도한 외형과 동작을 보여주는 프로토타입이며, 그대로 복사해 배포할 프로덕션 코드가 아닙니다.

작업 목표는 이 HTML 디자인을 **대상 코드베이스의 기존 환경(React/Vue/SwiftUI/네이티브 등)에서 그 패턴·라이브러리로 재구현**하는 것입니다. 환경이 없다면 프로젝트에 적합한 프레임워크를 선택해 구현하세요.

> 권장 아키텍처: **매칭 코어를 순수 함수로 분리** — `(참가자/상태, 코트 상황, 모드, 옵션) → 다음 경기 추천`. 이 번들의 `core.jsx`가 그 순수 함수 구현체이며, 서버(공용 API)로 옮겨 단위 테스트로 규칙 검증 후 앱·웹에서 호출 권장.

## Fidelity
**High-fidelity (hifi)** — 최종 색상·타이포·간격·인터랙션 확정. UI는 코드베이스의 기존 라이브러리/패턴으로 **픽셀에 가깝게 재현**하세요. 데이터(이름·급수·카운트)는 데모용 더미입니다.

플랫폼 우선순위: **① 태블릿(가로) 거치 메인 → ② 스마트폰(세로) → ③ 웹 반응형.** 현장에서 빠르게 탭하므로 터치 타겟 크게(주요 버튼 높이 ≥ 56px), 상태가 멀리서도 읽히게.

## 테마 — 토스풍 클린 (코트색 유지), 단일 확정
**확정 테마는 하나입니다: 토스/카카오식 미니멀 베이스 + 코트(종목)색 유지.** CSS 변수(`:root` + `[data-theme="clean"]` 오버라이드)로 구동되며, 인라인 스타일이 `var(--…)`를 참조합니다. 앱은 `theme: 'clean'`으로 고정되고 테마 토글 UI는 제거되었습니다.

성격:
- **베이스**: 흰 배경(`--bg:#fff`), 평평함(`--sh-1:none`, `--sh-2/3` 최소), 모달은 **바텀시트**(하단 슬라이드업 + 드래그 핸들).
- **코트색 유지**: 종목색(`--mix/men/women` 계열)·아바타 색은 `:root` 값을 그대로 상속. → 코트 카드 상단 **종목색 스트립**(진행 중 코트), **종목 배지 / 급수칩 / 미니코트 tint**에 색 포인트.
- 미니코트는 토스풍에 맞춰 dashed 코트라인·점선 네트 장식을 빼고 **얇은 회색 네트 + VS**만(색은 tint·배지·칩이 담당).

> `[data-theme="clean"]`은 중립/그림자 변수만 덮어쓰고 **종목·아바타 색 오버라이드는 두지 않습니다**(예전엔 무채색으로 죽였으나 제거). 종목색을 더/덜 강조하려면 `:root`의 `--mix/men/women-*` 값만 조정.

JS 분기(`useTheme`): 미니코트 네트 장식 on/off(`MiniCourt`), 모달 vs 바텀시트(`ModalShell`), 빈 코트 강조 틴트(`CourtCard`). 코트 상단 종목색 스트립은 모든 테마에서 표시(`strip=true`). 단일 테마라 분기 코드는 정리해도 무방.

## Interactions & Behavior

### 운영 플로우 (rolling / 코트별 비동기)
- 고정 R라운드 대진표가 **아님**. 코트 하나 빌 때마다 단발 계산.
- **경기 종료 = 원탭 사이클 ⭐**: `경기 종료` 누르면 → ① 4명 대기열 복귀(해당 종목 카운트 +1, `lastFinished=now`, 같은 팀 `partnerCount`++, 상대 `opponentCount`++) → ② **그 코트에 다음 추천 경기를 즉시 자동 배정** → ③ 호출 토스트. (설계 §8.5 "빈 코트 생기면 즉시 자동 확정")
- **되돌리기**: 토스트의 [되돌리기]가 종료 직전 스냅샷으로 복구.
- **추천 확정 = 자동, 편집 가능**: 운영자는 평소 종료만, 필요 시 [편집]으로 swap/종목 변경.
- 모드/성향 토글은 세션 중 언제든 → 이후 새로 채워지는 코트부터 적용, 진행 중 경기는 유지.

### 출석/상태
- 상태 3종: **미출석**(보류) / **참여중**(포함) / **퇴장**(제외). 모든 토글 양방향.
- 늦참 → 체크(경기수 0이라 최우선·"첫 경기" 플래그). 조퇴 → 퇴장. 노쇼 → 미출석.

### 공정성 가시화 (핵심 가치)
- 대기열: `totalGames===0` → **"첫 경기"** 뱃지(brand), `rest≥20분 & 대기` → **"오래 대기"** 뱃지(warn).
- 코트 카드: 진행 시간 "N분째"(타이머 트리거 아님, 정보용 — `now − match.startedAt`).

### 애니메이션
- 버튼 press translateY(1px)+그림자, transition .08s. 세그먼트 .18s. 바텀시트 sheetUp .3s. 토스트 slideUp.
- 카드 진입 애니메이션은 캡처/프린트 이슈로 미사용 — base가 곧 visible(opacity:1).

### 반응형
- 프로토타입은 고정 프레임(태블릿 1194×834, 폰 390×844)을 `transform: scale()`로 letterbox(`DeviceFrame`). **실제 앱에선 불필요** — 네이티브 반응형으로.

## Matchmaking Core (가장 중요 — `core.jsx`)

### 급수 → 점수
자강7 · S6 · A5 · B4 · C3 · D2 · 왕초심1. **혼복 보정(femaleAdjust=1)**: 혼복에서 여자 유효점수 = `max(1, 기본−1)`. 남복·여복 보정 없음.

### 대기열 정렬 (1차 공정성)
`① 경기수 적은 순 → ② 동률이면 쉰 시간 긴 순.` 연속 출전 가드 없음(경기수 +1로 자연 후순위).

### 선발 — "사람 먼저, 종목 그 다음"
대기열 앞쪽 윈도우(~9명)에서 후보를 모아, 현재 모드 안에서 가능한 종목을 구성. 모드는 절대 규칙이 아니라 **선호**.
- **혼복만**: 남≥1·여≥1이면 혼복(한쪽 부족 시 동성 페어 1팀 허용). 한쪽 완전 없음 → **혼복 불가 → 알림(자동으로 종목 안 바꿈, 운영자 선택)**.
- **남복·여복만**: 동성 4명 세트로 남복/여복.
- **전부 섞기**: 후보 성비로 자연스러운 종목 자동.

### 비용 함수 (낮을수록 좋음)
```
cost = w_balance  × (유효점수합차)²
     + w_partner  × (파트너 중복)
     + w_opponent × (상대 중복)
     + w_fairness × (네 명 누적 경기수 합) × 0.15
```
4명 → 두 팀 split 3가지 중 최소비용. **팀 나누기는 성향과 무관하게 항상 균형.** 윈도우 인덱스 합으로 대기열 앞 사람 우대(idxPenalty).

### 추천 근거 (`reason`)
`recommendMatch`가 반환: `"실력 합 동일" | "실력 차 ±N"` + `" · 새 조합" | " · 재호흡" | " · 리매치"`. 코트 카드·호출 토스트에 표시(알고리즘 신뢰 ↑).

### 성향 프리셋 (세션 중 토글, 기본 균형파)
| 성향 | w_balance | w_partner | w_opponent | w_fairness |
|---|---|---|---|---|
| **균형파**(기본) | 3 | 2 | 1 | 1 |
| **박빙파** | 5 | 1 | 0.5 | 1 |

## State Management
```ts
Participant {
  id, name, gender:'M'|'F', level:'자강'|'S'|'A'|'B'|'C'|'D'|'왕초심',
  status:'미출석'|'참여중'|'퇴장', court:number|null, team:'A'|'B'|null,
  games:{혼복,남복,여복}, totalGames, lastFinished:number|null,
  partnerCount:{[id]:n}, opponentCount:{[id]:n},
}
Court { no, match: { type, teamA:[p,p], teamB:[p,p], startedAt:number } | null, pendingRemove?:boolean }
Session { participants[], courts[], mode, preset, screen, device, theme, modal, toast, nowTs }
```
액션(`app.jsx`): `setMode/Preset/Theme/Device`, `setStatus(id,status)`, **`endMatch(court)`**(원탭 자동배정+토스트+undo 스냅샷; 예약제거 코트면 종료 시 제거), `startMatch`, `openEdit`, `confirmEdit`, `pickAlt`, `openCheckin`, `openSettings`, `setCourtCount(n)`(코트 수 1~8 증감), `toggleRemoveCourt(no)`(빈 코트 즉시 제거·경기 중은 pendingRemove 토글), `undo`, `dismissToast`.

### 코트 수 설정 (`SettingsModal`)
- 컨트롤 바 **⚙️ 코트 N면** → 시트에서 **스텝퍼(1~8면)** + 코트별 상태/개별 제거.
- **증설**: 빈 코트 추가 → 자동 추천이 채움. **감축**: 뒤 코트부터 — 빈 코트는 즉시 제거, **경기 중이면 `pendingRemove`로 표시 후 종료 시 자동 제거**(진행 경기 강제 중단 안 함). 추천/자동배정 계산은 `pendingRemove` 코트를 건너뜀.

### 권장 API (설계 §미정)
세션 시작 / 출석 토글 / 경기 종료(→다음 추천 반환) / 편집 / 모드·성향 토글 / 참가자 검색. 동시성은 "종료를 하나씩 누름 → 순서대로 한 코트씩" 자연 직렬화.

### 규모/성능
최대 50명/7~8코트(보통 30~40명/4~6코트). 추천은 코트 단위 단발 계산 — 후보를 대기열 앞쪽 윈도우로 제한하면 충분.
