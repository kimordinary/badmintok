# [앱] 출석 체크인 — 프로필 게이트 적용 요청

> 대상: 앱(프론트) 개발
> 서버는 적용 완료. 앱은 아래대로 화면/동작만 맞춰주면 됩니다.

## 한 줄 요약
출석 체크인을 하려면 프로필에 **실명·성별·급수**가 모두 있어야 합니다. 하나라도 비면 체크인 버튼을 막고 "프로필 먼저 채워주세요"로 안내해주세요.

---

## 1. 서버가 주는 신호

### (A) 내 상태 조회 응답에 `profile` 블록 추가됨
`GET /api/bands/match/<session_id>/me/`
`GET /api/bands/match/schedules/<schedule_id>/me/`

```json
{
  "...기존 필드 그대로...": "...",
  "profile": {
    "complete": false,
    "missing": ["name", "gender", "level"],
    "name": "",
    "gender": "unknown",
    "level": ""
  }
}
```

| 필드 | 의미 |
|------|------|
| `profile.complete` | `true`면 체크인 가능, `false`면 막아야 함 |
| `profile.missing` | 비어 있는 항목만. 값: `name`(실명) / `gender`(성별) / `level`(급수) |
| `profile.name/gender/level` | 현재 저장된 값 (표시·디버깅용) |

> 비로그인/예외 시 `profile`은 `null`일 수 있음 → `null`이면 막힌 것으로 처리(체크인 시도 시 서버가 422로 막음).

### (B) 체크인 API는 미완성이면 422로 거부함
`POST /api/bands/match/<session_id>/me/checkin/`  body: `{ "action": "in" }`

미완성 상태로 체크인 시도하면:
- HTTP **422**
```json
{
  "detail": "출석 체크인 전에 프로필(실명·성별·급수)을 먼저 입력해주세요.",
  "code": "profile_incomplete",
  "missing": ["name", "gender", "level"]
}
```
- `action: "out"`(퇴장)은 게이트 없음(항상 허용)

---

## 2. 앱이 해야 할 것

### 2-1. 체크인 화면 분기 (`profile.complete` 기준)

**`complete === true`** → [출석 체크인] 버튼 정상 노출

**`complete === false` (또는 `profile === null`)** → 버튼 대신 안내 카드:

```
┌────────────────────────────────────────┐
│ 출석 체크인하려면 프로필이 필요해요          │
│                                          │
│ 급수·성별·실명이 있어야 대진에 들어갈 수    │
│ 있어요. 프로필을 먼저 채워주세요.           │
│                                          │
│ [ 내정보 › 프로필 편집 ]                   │
└────────────────────────────────────────┘
```

색 토큰(웹과 동일):
| 요소 | 값 |
|------|-----|
| 카드 배경 | `#fdf4e3` |
| 카드 테두리 | `#f1e0bd` |
| 제목 | `#b87514` / 800 / 14px |
| 본문 | `#9a7a3c` / 13px |
| 버튼 bg/fg | `#b87514` / `#fff` |

(선택) `missing`으로 부족 항목만 콕 집어 안내해도 좋음:
- `["name"]` → "실명을 입력해주세요"
- `["gender","level"]` → "성별·급수를 입력해주세요"

### 2-2. 422 방어 처리 (필수)
화면 상태가 늦게 갱신돼 버튼이 눌릴 수 있으니, 체크인 POST 응답이 **422 `profile_incomplete`** 면 위 안내 카드/토스트로 동일 처리. `missing` 값 그대로 사용.

### 2-3. 프로필 편집 화면
- [내정보 › 프로필 편집]로 이동.
- **실명·성별·급수** 입력/수정 가능해야 함.
- 저장 후 체크인 화면 복귀 시 `me/` 재조회 → `complete=true`면 버튼 자동 노출.

> 프로필 편집 API 경로/필드는 서버 담당이 확정해 별도 전달(accounts 프로필 기준).

---

## 3. 코드값 참고
- 성별: `male` / `female` / `unknown`
- 급수: `master` / `s` / `a` / `b` / `c` / `d` / `beginner`
- 실명은 `Profile.name` 원본 기준(활동명/닉네임으로는 통과 안 됨)

---

## 4. 문구 (그대로 사용)
| 위치 | 문구 |
|------|------|
| 안내 카드 제목 | 출석 체크인하려면 프로필이 필요해요 |
| 안내 카드 본문 | 급수·성별·실명이 있어야 대진에 들어갈 수 있어요. 프로필을 먼저 채워주세요. |
| 버튼 | 내정보 › 프로필 편집 |
| 체크인 성공 토스트 | 출석 체크인 완료! 바로 대진에 반영돼요. |
| 퇴장 토스트 | 퇴장 처리됐어요. 대진에서 빠집니다. |

## 5. 앱 체크리스트
- [ ] `me/` 응답 `profile.complete`로 체크인 버튼 분기
- [ ] `complete=false`/`null` → 안내 카드 + [프로필 편집] 이동
- [ ] 체크인 POST `422 profile_incomplete` 방어 처리(`missing` 활용)
- [ ] 프로필 편집 화면에서 실명·성별·급수 입력 보장
- [ ] 저장 후 복귀 시 상태 재조회로 자동 해제 확인
