# 출석 체크인 — 프로필(실명·성별·급수) 게이트 핸드오프

## 0. 목적

참가자가 **출석 체크인**을 하려면 프로필에 **실명 · 성별 · 급수** 세 가지가 모두 채워져 있어야 한다.
하나라도 비어 있으면 체크인 버튼을 막고 "프로필을 먼저 채워주세요"로 안내한다.

> 자동 대진은 급수·성별로 팀을 짜고, 실명으로 코트에서 호명한다. 셋 중 하나라도 없으면 대진에 넣을 수 없으므로 **체크인 시점에서 차단**한다. (신청·승인 단계는 막지 않는다 — 체크인에서만 게이트)

웹에는 이미 게이트가 있으나(성별·급수만), **앱과 API에는 게이트가 없다.** 이 문서로 앱 + 서버 API를 통일한다.

---

## 1. 게이트 조건 (3개 모두 충족해야 통과)

| 항목 | 필드 | 통과 조건 |
|------|------|-----------|
| 실명 | `Profile.name` | 비어 있지 않음 (공백 제외) |
| 성별 | `Profile.gender` | `male` 또는 `female` (`unknown` 불가) |
| 급수 | `Profile.badminton_level` | 값이 있음 (빈 문자열 불가) |

```python
complete = bool(
    pf
    and pf.name and pf.name.strip()
    and pf.gender in ("male", "female")
    and pf.badminton_level
)
```

⚠️ **실명 주의**: `user.real_name`은 `Profile.name`이 비면 활동명(닉네임)으로 폴백한다. 게이트는 폴백 값이 아니라 **`Profile.name` 원본**을 검사해야 한다(활동명으로 때우는 것 방지).

급수 코드값: `master / s / a / b / c / d / beginner`
성별 코드값: `male / female / unknown`

## 2. 서버 API 스펙

### 2-1. (변경) 출석 체크인 — 게이트 추가
`POST /api/bands/match/<session_id>/me/checkin/`

요청:
```json
{ "action": "in" }   // in = 체크인, out = 퇴장
```

`action: "in"` 일 때 프로필 미완성이면 **체크인 거부**:

- 상태코드: `422 Unprocessable Entity`
- 응답:
```json
{
  "detail": "출석 체크인 전에 프로필(실명·성별·급수)을 먼저 입력해주세요.",
  "code": "profile_incomplete",
  "missing": ["name", "gender", "level"]
}
```
- `missing`: 비어 있는 항목만 배열로. 가능한 값 `name | gender | level`

`action: "out"`(퇴장)은 게이트 적용 안 함(항상 허용).

성공 시 기존과 동일하게 `my_status` 페이로드 반환(200).

### 2-2. (변경) 내 상태 조회 — 프로필 완성 여부 추가
앱이 **체크인 버튼을 미리 막으려면** 상태 응답에 프로필 완성 여부가 있어야 한다. 아래 두 엔드포인트 응답에 `profile` 블록 추가.

- `GET /api/bands/match/<session_id>/me/`
- `GET /api/bands/match/schedules/<schedule_id>/me/`

추가 필드:
```json
{
  "...기존 필드...": "...",
  "profile": {
    "complete": false,
    "missing": ["name", "gender", "level"],
    "name": "",
    "gender": "unknown",
    "level": ""
  }
}
```

### 2-3. (참고) 프로필 조회·수정 API
앱의 "프로필 편집" 화면이 쓰는 기존 엔드포인트. 실명·성별·급수 입력/저장이 가능한지 확인 필요.
> 서버 담당이 정확한 경로/필드명 채워서 전달 예정 — accounts 프로필 API 기준.

## 3. 앱 화면 / 동작 요청

### 3-1. 출석 체크인 화면 (참가자 본인)
체크인 버튼을 누르기 전, `profile.complete` 값으로 분기.

**`complete === true`**
- [출석 체크인] 버튼 정상 노출 → 누르면 `POST .../me/checkin/ {action:"in"}`

**`complete === false`**
- 체크인 버튼 자리에 **안내 카드** 노출(버튼 비활성/대체):

```
┌────────────────────────────────────────┐
│ 출석 체크인하려면 프로필이 필요해요          │   ← 14px / 800 / #b87514
│                                          │
│ 급수·성별·실명이 있어야 대진에 들어갈 수    │   ← 13px / #9a7a3c
│ 있어요. 프로필을 먼저 채워주세요.           │
│                                          │
│ [ 내정보 › 프로필 편집 ]                   │   ← #b87514 버튼 → 프로필 편집 이동
└────────────────────────────────────────┘
```
색 토큰: 배경 `#fdf4e3` / 테두리 `#f1e0bd` / 제목 `#b87514` / 본문 `#9a7a3c` (웹과 동일)

- `missing` 배열로 부족 항목만 콕 집어 줄 수도 있음(선택):
  - 예) `실명, 급수`만 비면 → "실명·급수를 입력해주세요"

### 3-2. 방어 로직 (중요)
- 화면에서 막아도 **서버가 422를 줄 수 있다**(상태가 늦게 갱신된 경우 등). 체크인 POST가 `422 profile_incomplete`로 오면 위 안내 카드/토스트로 동일하게 처리.
- 422의 `missing`을 그대로 사용하면 일관됨.

### 3-3. 프로필 편집 화면
- 실명(`name`) · 성별(`gender`) · 급수(`badminton_level`) 입력/수정 가능해야 함.
- 저장 후 체크인 화면으로 돌아오면 상태 재조회(`me/` 폴링)로 자동 해제.

## 4. (선택·후속) 번개 당일 일괄 알림
승인됐지만 프로필 미완성인 참가자에게 당일 푸시로 "프로필 채워달라" 유도.
- 기존 FCM 인프라 사용: `notifications/push.py` `send_to_users()`
- 타겟: 해당 번개 `approved` 신청자 중 게이트 미충족자
- 발송: 자동(당일 스케줄) vs 모임장 수동 버튼 — **미정**(별도 협의)
- 본 핸드오프(체크인 게이트)와 분리해서 후속 작업으로 진행 가능.

| API 거부 detail | 출석 체크인 전에 프로필(실명·성별·급수)을 먼저 입력해주세요. |
